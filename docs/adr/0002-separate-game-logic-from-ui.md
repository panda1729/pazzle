# ADR-0002: ゲームロジックを UI 非依存の純粋関数に分離する

- Status: Accepted
- Date: 2026-07-19

## Context

プロトタイプでは迷路生成・経路探索・スコア計算・描画・状態管理が 1 ファイルに同居しており、ギミック追加(一筆書きモード、回数制限マス等)のたびに全体を触ることになる。ロジックの正しさをテストで担保する手段もなかった。

## Decision

レイヤーを次のように分離する。

- `src/game/`: 迷路生成・ソルバー・スコア・ステージ定義。**React にも DOM にも依存しない純粋関数**とし、ユニットテストの対象はここに集中させる。
- `src/hooks/useGame.ts`: ゲーム進行の状態管理。useReducer で「移動」「リセット」等をアクションとして表現する。
- `src/components/`: 描画と入力のみ。ロジックを持たない。

あわせて、ステージの par(最短手数)と limit は手書きせず、ソルバーの結果から自動算出する。

## Consequences

- ギミック追加は「game 層に純粋関数を足す → reducer に組み込む → 描画を足す」という定型の手順になる。
- par/limit の手書き値と実際の迷路の乖離が構造的に起きない。ステージを追加・変更してもクリア可能性がビルド時に検証される(到達不能ならエラー)。
- 将来マップ自動生成を入れる場合も、生成 → ソルバーで検証 → par 算出のパイプラインをそのまま流用できる。
