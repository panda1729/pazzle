# ADR-0001: Vite + React + TypeScript を採用する

- Status: Accepted
- Date: 2026-07-19

## Context

プロトタイプは単一の JSX ファイル(React 前提)として作られていた。これを個人開発として継続できるプロジェクト構成に移行するにあたり、ビルドツール・言語・テスト基盤を決める必要があった。

- 2D 迷路パズルのブラウザゲームで、サーバーサイド処理は不要(静的サイトで完結)
- 開発者は一人。セットアップとメンテナンスのコストは低く保ちたい

## Decision

- **React 19 + TypeScript**: プロトタイプが React 製で流用しやすい。型があることでゲームロジック(グリッド・座標・ステージ定義)の間違いをコンパイル時に検出できる。
- **Vite**: 設定がほぼ不要で開発サーバーが速い。静的ビルドがそのままホスティングに使える。
- **Vitest**: Vite と設定を共有でき、ゲームロジックのユニットテストを追加コストなしで書ける。
- **ESLint**(typescript-eslint + react-hooks): 標準的な React + TS 構成の lint。

Next.js 等のフレームワークは、SSR・ルーティングが不要なため採用しない。

## Consequences

- 静的サイトとしてビルドされるため、無料の静的ホスティング(GitHub Pages 等)にそのまま公開できる(ADR-0003)。
- スコア保存などの永続化はクライアントサイド(localStorage 等)で行う前提になる。サーバーが必要になったらその時点で ADR を追加する。
